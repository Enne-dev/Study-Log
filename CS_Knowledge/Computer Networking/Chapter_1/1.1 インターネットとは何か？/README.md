上記の質問に答える方法としては、以下の2つの視点が存在する。

1.1.1 構成要素から見たインターネット
下の図はインターネットの構成要素を表したものである。

インターネット (Internet)

Network of Networks

全世界で数十億個のコンピューティングデバイスを接続するコンピュータネットワーク。

ホスト (host)、エンドシステム (end system)

コンピュータネットワークに接続されたコンピューティングデバイス。

e.g., サーバ（デスクトップPC、Linuxワークステーション、Webページなど）、インターネットに接続されたあらゆる「モノ」（TV、スマートウォッチなど）。

通信リンク (communication link) とパケットスイッチ (packet switch) のネットワークで接続される。

通信リンク (communication link)

様々な転送レート (transmission rate、リンク帯域幅またはbandwidth) を利用してパケット (packet = データ) を転送する。

転送レートの単位：bps (bits per second、1秒あたりのビット数)。

同軸ケーブル、銅線、光ファイバー、無線スペクトルを含む様々な物理媒体で構成される。

パケット (packet)

送信側エンドシステムから受信側エンドシステム（宛先）へ送られる。

送信側エンドシステムが、送ろうとするデータをセグメント (segment) に分割し、各セグメントにヘッダ (header) を付加して受信側エンドシステムへ転送する。

パケットは、宛先で元のデータに再構成（リアセンブル）される。

パケット交換機、パケットスイッチ (packet switch)

入力通信リンクの中の一つ。

到着するパケットを受け取り、出力通信リンクの一つへそのパケットを転送する。（最終的な宛先の方向へパケットを転送）

代表的な2種類：

リンク層スイッチ (link-layer switch)：主にアクセスネットワークで使用。

ルータ (router)：主にネットワークコアで使用。

経路 (route, path)

パケットが送信側エンドシステムから送出された後、受信側エンドシステムに到達するまでに通過する一連の通信リンクとパケットスイッチのこと。

パケットはコンピュータネットワーク上の経路に従う。

ISP (Internet Service Provider)

パケットスイッチと通信リンクで構成されるネットワーク。

エンドシステムに様々なネットワーク接続を提供する。（家庭用ブロードバンド接続、高速LAN接続、モバイル無線接続など）

CP (Content Provider) にインターネット接続を提供する → Webサイトやビデオサーバをインターネットに直接接続できるようになる。

ISPの相互接続

💡 インターネットはエンドシステム同士を接続するものであるため、エンドシステムに接続を提供するISP同士も接続されていなければならない。

下位層ISPは、国家および国際的な上位層ISPを通じて相互接続する。（上位層ISP同士は直接接続される）

各ISPネットワークは個別に管理され、IPプロトコルを実行し、ネーミング (naming) とアドレス割り当て方式に従う。

プロトコル (protocol)

インターネットにおける情報の送受信を制御する。

最も重要な2つのプロトコルを総称してTCP/IPと呼ぶ。

TCP (Transmission Control Protocol)

IP (Internet Protocol)：ルータとエンドシステム間で送受信されるパケットのフォーマットを記述する。

標準 (Standards)

IETF (Internet Engineering Task Force)：国際インターネット標準化機構。

RFC (Requests for Comment)：IETF標準文書。

TCP, IP, HTTP, SMTPなどのプロトコルを定義。

IEEE 802 LAN標準委員会：イーサネットと無線Wi-Fiの標準を記述。

1.1.2 サービス側面から見たインターネット
インターネットはアプリケーションにサービスを提供するインフラストラクチャ (Infrastructure) である。

アプリケーションは互いにデータを交換する多くのエンドシステムを含んでいるため、分散アプリケーション (distributed application) と呼ばれる。

インターネットアプリケーションはエンドシステム上で実行され、ネットワークコアにあるパケット交換機では実行されない。

💡 パケット交換機はエンドシステム間のデータ交換を容易にするが、データの始点と終点であるアプリケーションには関与しない。

ソケットインターフェース (socket interface)

Q. あるエンドシステムで実行されるアプリケーションが、他のエンドシステムで実行されているプログラムにデータを送るようにするためには、インターネットにどのように指示すればよいか？

あるエンドシステムで実行されるプログラムが、他のエンドシステムで実行される特定の宛先プログラムへデータを伝達するように、インターネット・インフラストラクチャに対してどのように要求するかを明示したものをソケットインターフェースという。

→ インターネットに接続されたエンドシステムは、すべてソケットインターフェースを持っている。

💡 ソケットインターフェースは、送信プログラムが従うべき規則の集合であり、インターネットはこの規則に従ってデータを宛先プログラムへ伝達することになる。

1.1.3 プロトコルとは何か？
2つ以上の通信エンティティ (entity) が何らかの作業を共同で遂行するには、それらが共に認識するプロトコル、すなわち通信規約が必要である。 下の図は、人間のプロトコルとコンピュータネットワークのプロトコルを表したものである。

メッセージの送受信と、メッセージを送受信する際に取る行動は、プロトコルの中心にある。

したがって、一方が異なるプロトコルを実行している場合、そのプロトコルは他者と相互作用できず、目的の作業を遂行することができない。

ネットワークプロトコル

通信する2つ以上のリモートエンティティが含まれるインターネット上のすべての活動は、プロトコルが制御する。

e.g.,

輻輳制御 (congestion-control) プロトコル：エンドシステムに存在し、送受信者間で転送されるパケットの転送レートを調節する。

ルータにおけるプロトコル：送信元 (source) から宛先 (destination) までのパケット経路を設定する。

💡 プロトコルは、2つ以上の通信エンティティ間で交換されるメッセージのフォーマットと順序だけでなく、メッセージの送受信やその他のイベントに応じた行動を定義する。
